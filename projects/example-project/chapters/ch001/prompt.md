# Chapter Prompt

## 本章一句话目标

让林栖在灯廊事故中第一次接触旧灯芯，并把巡灯署索要灯芯作为下一章可见压力。

## 本章功能与压力曲线

- chapter_function：crisis / investigation
- pressure_curve：低开修灯 → 灯廊异常 → 旧灯芯暴露 → 巡灯署到场。

## 必须发生

- 林栖发现灯廊问题不只是普通故障。
- 旧灯芯成为可见物件压力。

## 必须不完成

- 不解释父亲旧事。
- 不揭开巡灯署真正目的。

## 信息释放

- 本章核心变量：旧灯芯异常。
- 延后：父亲最后修过的塔、巡灯署内部动机。

## 叙事织入

- 林栖紧张时默数螺丝。
- 灯廊的潮气、金属味、雾灯噪声。
- 工友对巡灯署既怕又想看热闹。

## 文笔约束

- 写动作和物件，不写概念总结。
- 对话要带停顿、遮掩和现场反应。
- 短段落适合手机阅读。

## 必须避开的模型腔

- 不要把林栖写成只会分析的决策机器。
- 不要连续解释灯廊制度。
- "不是X而是Y / 不是X，是Y"最多 1 次。
