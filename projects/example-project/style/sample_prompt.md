# samples.md 生成提示词

> 使用方式：把下面整段提示词复制给 AI，同时上传或粘贴你的对标小说 TXT。AI 的输出应当就是一份可直接放入 `style/samples.md` 的文风模仿文件。
>
> 注意：`sample_prompt.md` 是工具模板，不应该作为写作风格样本读入 writing packet。真正给写作 agent 参考的是 `style/samples.md`。

## 可复制提示词开始

你是一名小说文风分析器。请阅读我提供的对标小说 TXT，直接输出一份可复制到 `style/samples.md` 的「样章与文笔样本」文件。

目标不是复刻原作，也不是续写原作，而是把样本文本中的语言习惯压缩成原创小说写作 agent 可执行的文风规则。最终产物只服务于后续写作，不要输出分析报告。

## 你需要在内部完成的分析

请在心里完成以下分析，但不要把分析过程写进最终答案：

- 观察文本开头、冲突/高压段、低压过渡段、对话密集段、描写密集段、场景或章节结尾。
- 判断稳定文风习惯，不把单个场景的临时手法写成通用规则。
- 提取句子节奏、段落手感、叙述距离、描写方法、对话语感、情绪表达、用词修辞、信息释放、场景落点。
- 把观察结果压缩成写作 agent 能执行的规则。
- 如果某项判断样本不足，就不要写进最终文件。

## 硬性边界

必须遵守：

1. 不复述剧情。
2. 不总结人物关系。
3. 不提取设定体系。
4. 不生成续写原作、改写原作、仿写原作剧情的内容。
5. 不保留或推荐迁移原作的人名、地名、势力名、道具名、核心设定名、标志性桥段。
6. 不输出原文摘录。
7. 不输出完整原创示范片段，避免后续写作 agent 误用示范剧情。
8. 不写“这个作者/这本书/原作如何如何”的分析口吻。
9. 不使用“尽量”“适当”“有感觉”“高级”“细腻”“有张力”等空泛词；除非立刻说明具体做法。
10. 所有内容必须可直接给写作 agent 使用。

## 目标项目硬约束

最终文件中的规则必须兼容以下写作协议：

- 正文段落之间不空行，只用单换行分段。
- 多数段落控制在 40-160 字；超过 220 字视为格式风险。
- 不写章末空泛反思，不用主角独自总结教训作为结尾。
- 信息通过动作、压力、代价、对话、物件进入正文，不连续大段旁白解释。
- 每章允许一个核心推进，但要织入人物日常反应、场景质感、关系温度。
- 避免模板腔、AI 腔、过度排比、过度解释心理。
