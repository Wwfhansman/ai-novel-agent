# 工作流设计

## 1. Skill 划分

MVP 使用 4 个核心 skill：

```text
novel-bootstrap
novel-write
novel-review
novel-change
```

## 2. Novel Bootstrap

### 2.1 使用场景

用于：

- 新项目为空。
- 用户明确要开新书。
- 用户明确要求重启或重构作品 DNA。

不用于：

- 中途加入普通点子。
- 修改人物关系。
- 调整局部设定。
- 修改当前卷节奏。

### 2.2 输入

用户给出创作 seed：

- 一句话设定
- 主角形象
- 世界观点子
- 爽点
- 开篇场景
- 类型方向

### 2.3 输出

- 创作宪法
- 读者模型
- 风格记忆
- 全书初始摘要
- 第一卷卷纲
- 初始实体库
- 初始动态账本
- 当前连续剧情流 active_flow
- 近期 9-15 章详细章纲

### 2.4 流程

```text
读取用户 seed
→ 生成 3-5 个创作方向
→ 评估长篇潜力和类型承诺
→ 选择或推荐方向
→ 建立创作宪法
→ 创建初始故事圣经
→ 创建核心人物和势力
→ 创建第一卷目标
→ 创建第一条连续剧情流
→ 创建近期详细章纲
→ 初始化项目文件
```

## 3. Novel Write

### 3.1 使用场景

用于日常写作。

默认一轮写 3 章。

### 3.2 一轮开始流程

```text
读取创作宪法
读取读者模型
读取全书摘要
读取已完成卷摘要
读取当前卷纲和状态
读取当前 arc
读取最近 12-15 章摘要
按连续性需要读取最近 1-3 章全文
按本批次涉及对象定向读取动态账本
读取并刷新 active_flow 连续剧情流
全文读取并刷新 rolling_plan 详细章纲
根据本批次相关压力和债务回看关键旧章节
生成 round context_pack.md
生成前文理解报告
从 rolling_plan 摘取本批次 3 章到 current_round
```

### 3.3 单章流程

```text
读取本章 brief
→ 读取 active_flow
→ 从已全文读取的 rolling_plan 中摘取本章、相邻章节和必要后续约束
→ 检查上一章 actual_handoff
→ 生成 chapter context_pack.md
→ 生成本章理解
→ 确认本章叙事债、伏笔、信息可见性、角色意图、世界压力
→ 可选生成自由写作 notes
→ 写 draft.txt
→ 自审
→ 修改为 final.txt
→ 生成 summary.yml
→ 生成 canon_delta.yml
→ 更新实体和动态账本
→ 更新 active_flow
→ 生成 actual_handoff，并同步 active_flow.last_cut.current_handoff
```

### 3.4 一轮结束流程

```text
汇总本批次变化
→ 更新当前卷摘要和状态
→ 更新当前 arc
→ 保持 active_flow 跨过 round 边界，除非故事自然抵达 payoff
→ 检查叙事债务
→ 检查信息可见性
→ 检查世界状态
→ 评估新灵感
→ 滑动刷新后续 6-15 章详细章纲
→ 记录本轮 diff 摘要
→ 写入 session log
```

## 4. Novel Review

### 4.1 使用场景

用于：

- 每轮 3 章结束后的冷启动审查。
- 用户要求检查质量。
- 长篇出现跑偏迹象。
- 写作前确认项目记忆是否足够。

### 4.2 审查维度

- 本书是否仍符合创作宪法。
- 当前卷是否仍朝目标推进。
- 最近章节是否有追读动力。
- 叙事债是否拖太久。
- 人物行为是否符合角色意图。
- 信息可见性是否穿帮。
- 世界是否对主角行动持续反应。
- 伏笔是否被遗忘。
- 风格是否漂移。
- 项目文件是否足以让新 agent 接手。

### 4.3 输出

- 问题列表
- 风险等级
- 修改建议
- 是否建议重写章节
- 是否建议进入 novel-change

## 5. Novel Change

### 5.1 使用场景

用于处理中途新想法或变更。

例如：

- 用户想加入一个结尾反转。
- 用户想改变配角定位。
- 用户想调整第一卷主线。
- 用户想修改某个设定。

### 5.2 变更分级

```text
Level 1: 风格变更
Level 2: 局部设定变更
Level 3: 人物/关系变更
Level 4: 主线/卷纲变更
Level 5: 作品 DNA 变更
```

### 5.3 流程

```text
读取用户变更
→ 判断影响级别
→ 读取相关 canon
→ 检查冲突
→ 提供接入方案
→ 判断是否进入 idea_pool、foreshadowing、outline 或 canon
→ 更新相关文件
→ 标记需要补铺垫或重写的章节
```

### 5.4 原则

中途点子不默认重启项目。

只有 Level 5，或者用户明确要求重开，才升级到 Bootstrap。

## 6. 灵感处理流程

写作、审稿和复盘时，agent 应捕捉 Creative Sparks。

```text
发现未来可能
→ 写入 idea_pool
→ 标记状态 candidate 或 incubating
→ 每轮结束评估
→ 少量晋升为伏笔或规划
→ 不适合的标记 rejected 或 expired
```

## 7. 重大决策门禁

以下事项不能由 `novel-write` 静默执行：

- 杀死主要角色
- 确认或终止主要感情线
- 揭露终局秘密
- 改变主角核心动机
- 改变当前卷目标
- 推翻创作宪法
- 大规模 retcon

遇到这些事项时，应进入 `novel-change` 或请求用户确认。

受保护文件修改前，必须输出 diff 摘要并建立 checkpoint。详细规则见 [正史与安全规则](CANON_AND_SAFETY.md)。
