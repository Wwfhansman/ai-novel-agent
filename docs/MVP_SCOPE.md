# MVP 边界

## 1. MVP 目标

MVP 的目标不是做完整商业产品，而是验证核心假设：

> 基于文件化结构记忆和 agent 工作流，AI 是否能稳定创作连续发展的长篇小说。

## 2. MVP 包含内容

### 2.1 四个核心 skill

```text
novel-bootstrap
novel-write
novel-review
novel-change
```

### 2.2 文件化项目结构

每本小说是一个独立项目目录，包含：

- 全书记忆
- 卷记忆
- 章群记忆
- 章节正文和章节状态
- 实体库
- 动态账本
- 滚动规划
- 风格记忆
- 元信息

### 2.3 三章一轮写作

默认创作节奏：

```text
一轮写 3 章
每章独立规划、写作、审稿、更新
每轮结束复盘并调整后续规划
```

### 2.4 分层上下文编译

写作前按层读取：

- 创作宪法
- 全书摘要
- 已完成卷摘要
- 当前卷纲和状态
- 当前 arc
- 最近 12-15 章摘要
- 最近 3-5 章全文
- 关键旧章节原文
- 动态账本
- 本章任务

每次编译必须产出标准化 `context_pack.md`。MVP 可以不写自动编译程序，但不能省略 context pack artifact。

### 2.5 动态账本

必须支持：

- 读者模型
- 叙事债务
- 信息可见性
- 角色意图
- 世界状态
- 伏笔
- 灵感池

### 2.6 变更管理

支持用户中途加入点子、修改设定、调整人物关系和卷纲。

变更管理不默认重启全书。

## 3. MVP 不包含内容

暂不开发：

- Web UI
- 桌面客户端
- 用户账号
- 云端数据库
- 向量数据库
- 多 agent 并发
- 自动发布到小说平台
- 评论数据分析
- 商业化支付
- 权限协作系统
- 复杂可视化看板

注意：上下文编译器的自动化代码可以暂不做，但 context pack 模板和人工/agent 生成流程属于 MVP。

## 4. 第一轮实验目标

用一个测试 seed 跑通：

```text
Bootstrap 初始化
→ 第 1 轮 3 章
→ Review 冷启动检查
→ 第 2 轮 3 章
→ 用户插入一个新点子
→ Change 接入
→ 持续写到 30 章
```

## 5. MVP 成功指标

### 5.1 创作连续性

写到 30 章后：

- 主角目标清楚。
- 当前卷目标未跑偏。
- 重要人物状态稳定。
- 世界状态有连续反应。
- 信息可见性无明显穿帮。

### 5.2 追读逻辑

系统能记录和处理：

- 羞辱反击债
- 复仇债
- 谜题债
- 承诺兑现债
- 势力扩张债

并能在合理窗口推进或偿还。

### 5.3 冷启动能力

新会话读取项目文件后，能回答：

- 当前故事讲到哪里。
- 主要人物现在想要什么。
- 哪些债务最紧迫。
- 哪些秘密谁知道。
- 下一轮三章应该做什么。

### 5.4 变更吸收能力

用户中途提出新点子后，系统能：

- 判断影响级别。
- 提供接入方案。
- 更新灵感池、伏笔、规划或状态。
- 不破坏既有 canon。

## 6. 超出 MVP 后再考虑

MVP 跑通后，再考虑：

- 将文件结构映射到数据库。
- 做本地工作台。
- 做上下文编译脚本。
- 做 schema 校验。
- 做章节导出和发布格式。
- 做读者反馈接入。
- 做多模型适配。
